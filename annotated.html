[detail level 12]
 ▼Neftcamb_abstract_modelThis module contains the abstract definition of all the places where EFTCAMB interacts with CAMB. All EFTCAMB models should inherit from this class or the two derived classes contained in 05p2_abstract_EFTCAMB_full.f90 or 05p3_abstract_EFTCAMB_designer.f90
 Ceftcamb_modelThis is the abstract type for EFTCAMB models. As a rule, when there is a new model it should be declared as a class inheriting from EFTCAMB_model. This guarantees maximum performances as well as maximum flexibility
 CEFTCAMBModelAllocateModelSelectionSubroutine that reads the parameters of the model from file
 CEFTCAMBModelBackgroundEFTFunctionsSubroutine that computes the value of the background EFT functions at a given time. The interface of the function is slightly complicated for performance reasons
 CEFTCAMBModelComputeAdotoaSubroutine that computes adotoa = H and its two derivatives wrt conformal time. Again the interface is slightly complicated for performance reasons
 CEFTCAMBModelComputeDtaudaFunction that computes dtauda = 1/sqrt(a^2H^2). Again the interface is slightly complicated for performance reasons
 CEFTCAMBModelComputeParametersNumberSubroutine that computes the number of parameters of the model
 CEFTCAMBModelFeedbackSubroutine that prints on the screen feedback information about the model
 CEFTCAMBModelInitModelParametersSubroutine that reads the parameters of the model from file
 CEFTCAMBModelInitModelParametersFromFileSubroutine that reads the parameters of the model from file
 CEFTCAMBModelParameterNamesSubroutine that returns the i-th parameter name of the model
 CEFTCAMBModelParameterNamesLatexSubroutine that returns the i-th parameter name of the model
 CEFTCAMBModelParameterValuesSubroutine that returns the i-th parameter name of the model
 CEFTCAMBModelReadModelSelectionFromFileSubroutine that reads the parameters of the model from file
 CEFTCAMBModelSecondOrderEFTFunctionsSubroutine that computes the value of the second order EFT functions at a given time. The interface of the function is slightly complicated for performance reasons
 ▼Neftcamb_abstract_model_designerThis module contains the abstract definition of all the places where EFTCAMB interacts with CAMB in case of a designer model. All models implementing a model in which the cosmological background is parametrized according to some choice of the user should inherit from this class
 Ceftcamb_designer_modelThis is the abstract type for EFTCAMB models
 ▼Neftcamb_abstract_model_fullThis module contains the abstract definition of all the places where EFTCAMB interacts with CAMB in case of a full mapping model. All models implementing a model in which the cosmological background is computed from the values of the EFT functions should inherit from this class
 Ceftcamb_full_modelThis is the abstract type for EFTCAMB models
 ▼Neftcamb_abstract_parametrizations_1dThis module contains the abstract class for generic parametrizations for 1D functions that are used by several models in EFTCAMB. As a rule, when there is a free function in EFT it should be declared as a class inheriting from parametrized_function
 Cparametrized_function_1dThis is the abstract type for parametrized functions. As a rule, when there is a free function in EFT it should be declared as a class inheriting from parametrized_function. This guarantees maximum performances as well as maximum flexibility
 ▼Neftcamb_constant_parametrization_1dThis module contains the definition of the constant parametrization, inheriting from parametrized_function_1D
 Cconstant_parametrization_1dType containing the constant function parametrization. Inherits from parametrized_function_1D. Notice that the derivatives are not overridden since they are zero identically
 ▼Neftcamb_linear_parametrizations_1dThis module contains the definition of the linear parametrization, inheriting from parametrized_function_1D
 Clinear_parametrization_1dType containing the linear function parametrization. Inherits from parametrized_function_1D. Notice that the derivatives above the first are not overridden since they are zero identically
 ▼Neftcamb_mainThis module contains the general EFTCAMB driver. We have a type that encapsulate all EFTCAMB parameters and stuff and this is owned by CAMB to do all EFT calculations
 CeftcambThis is the main object for EFTCAMB. Get one of these and you can use all the stuff in EFTCAMB
 ▼Neftcamb_pure_eft_stdThis module contains the definition of the Pure EFT model in which the EFT is described by six functions of time and w_DE. Please refer to the numerical notes for details
 Ceftcamb_std_pure_eftThis is the pure EFT model with the six functions of time and w_DE
